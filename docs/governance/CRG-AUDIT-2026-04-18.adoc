== Scope and Standard

Evaluates the `verification-ecosystem/session-sentinel` repository against
*Component Readiness Grades v2.0 (STRICT)*, 2026-03-30 revision.
session-sentinel is self-declared *C* via a sectioned `[crg] grade = "C"`
block in STATE.a2ml (achieved 2026-04-04). The repo is categorised as a
*container/infrastructure* repo; its self-declared C claim is explicitly
tied to *structural* test coverage (Deno-based file/TOML validation) rather
than to a running daemon binary. The audit checks the claim against
on-disk evidence, estate-wide consumer signal, and the v2.0
"honest D > dishonest B" clause.

- Standard: `standards/component-readiness-grades/COMPONENT-READINESS-GRADES.md`
- Criteria boilerplate: `docs/governance/CRG-CRITERIA.adoc` (already in-tree)
- Audit template: `rsr-template-repo/docs/governance/CRG-AUDIT-TEMPLATE.adoc`
- Calibration sibling: `verification-ecosystem/proof-of-work/docs/governance/CRG-AUDIT-2026-04-18.adoc`
- Self-declared grade (prior art):
  * `.machine_readable/6a2/STATE.a2ml` `[crg] grade = "C"`,
    `achieved = "2026-04-04"`.
  * `[project-context] completion-percentage = 40`.
  * `[route-to-mvp]` lists Phase 3 (Rust source) at 10% and Phase 4
    (CI/CD) at 20%.
  * `[blockers-and-issues]` concedes: "Rust source implementation
    incomplete — Phase 3 not started", "CI/CD not wired to deployment".
- Audit date: 2026-04-18
- Auditor: Jonathan D.A. Jewell

Schema note: session-sentinel uses the *sectioned* `[crg] / grade = "C"`
form (one of the 16 sectioned-schema repos in the Gossamer alignment
work). It does *not* carry a flat `[metadata] crg-grade = "C"` key, so
the current HYP-S005 detector (keyed on the flat form) will *not* fire
on this repo despite the overclaim documented below. See
"HYP-S005 canary" section.

The audit grades the repo *as-is today*, not aspirationally.

=== HYP-S005 canary

HYP-S005 is the estate-wide detector for
"self-declared C while evidence supports E or D". Its existing match
rule keys on the *flat* `crg-grade = "C"` key under `[metadata]`.
session-sentinel uses the *sectioned* schema:

[source,toml]
----
[crg]
grade = "C"
achieved = "2026-04-04"
evidence = [ ... ]
----

with *no* flat `crg-grade = "C"` key. The current HYP-S005 regex will
therefore *not* fire on this repo despite the overclaim documented here.

This is a **rule-drift finding**: session-sentinel is one of the 16
sectioned-schema repos in the Gossamer alignment inventory; those repos
are currently a blind spot for the flat-schema HYP-S005 rule. The rule
should be extended to match both forms:

- flat: `^crg-grade\s*=\s*"C"` under `[metadata]`
- sectioned: `^\[crg\]` followed by `^grade\s*=\s*"C"` within the
  section

Three independent overclaim signals would trigger the extended
HYP-S005 on session-sentinel:

1. Declared CI-green evidence: the 22 Deno tests *are not run by any
   CI workflow* (no `deno`/`zig`/`cargo` invocation in any YAML under
   `.github/workflows/`).
2. Declared dogfooding: the repo's own `[maintenance-status]` says
   `last-run-utc = "never"` and `[blockers-and-issues]` says "Rust
   source implementation incomplete — Phase 3 not started".
3. File-Map drift: EXPLAINME claims `src/panels/manifest.json`,
   `verification/proofs/*.idr`, `src/interface/generated/abi/*.h`
   which do not exist on disk.

No HYP-S005 suppression comment is present in the repo and none is
warranted.

== Verdict

*Current CRG grade: **D (Alpha)**. Demote one grade from self-declared C.*

Self-declared: *C*. Audited: *D*. One-grade honesty delta — identical to
proof-of-work, smaller than neurophone's two-grade delta. No
committed merge-conflict markers or fabricated EXPLAINME here; the
defect is subtler — *annotation discipline is real, but it annotates
software that does not run in the declared pipeline*.

Reasoning (one sentence): session-sentinel has the cleanest annotation
and doc-tree discipline of any recent C-candidate, *and* the 22 declared
tests match grep exactly, *but* it fails the C clauses on three
independent axes — (a) no `[dogfooding-status]` section and the repo's
own `[maintenance-status]` concedes `last-run-utc = "never"`, (b) the
declared 22 Deno tests are not invoked by any workflow under
`.github/workflows/`, (c) ~8 000 LOC of source across AffineScript /
Idris2 / Zig is written but not typechecked, built, or run by CI.

The D grade is earned, not conservative-by-default, on three positive
signals:

1. *Annotation depth*: 19/19 source files SPDX-clean with substantive
   module doc blocks; 17-subtree `docs/` tree including populated
   governance sub-subtree (CRG-CRITERIA, MAINTENANCE-CHECKLIST,
   SOFTWARE-DEVELOPMENT-APPROACH, TSDM); per-directory README coverage
   near 100%. This is the highest annotation score in the recent
   C-audit cohort.
2. *Test-count honesty*: declared 22 tests match grep exactly (seven
   labelled categories, seven matching `Deno.test(...)` label groups —
   no overclaim). Contrast with proof-of-work's 37-56% overclaims.
3. *Architectural intent is internally coherent*: Idris2 ABI
   (HealthState.idr with `%default total` and typed lattice of zones)
   + Zig FFI (StatusNotifierItem DBus) + AffineScript core (affine-typed
   scanner/healer/monitor loop) is a defensible ABI/FFI standard
   instantiation. 2 050 LOC of Idris2 + 2 487 LOC of Zig + 3 555 LOC
   of AffineScript is substantial written work.

What blocks *immediate* promotion to C (do these first):

. **Wire CI to run the declared tests.** Add a workflow step
  `deno test --allow-read tests/` to one of the existing workflows (or
  a dedicated `tests.yml`) so the 22-test structural harness runs on
  every push. Without this, the C claim has no CI backing.
. **Add `[dogfooding-status]` to STATE.a2ml** with concrete done-entries.
  Candidates: "session-sentinel-tray installed via systemd/install.sh on
  author's Fedora 43 system, running since DATE; last 24h scan log at
  PATH". Or *remove the C claim* until there is real use.
. **Smoke-test the systemd install.** ROADMAP v0.1.1 already lists this
  as an unchecked item. Run `systemd/install.sh`, let the tray run for
  24h, capture a scan log, cite in STATE. That alone lifts the
  `last-run-utc = "never"` blocker.
. **Reconcile EXPLAINME File Map with on-disk truth.** Either create the
  missing artefacts (`src/panels/manifest.json`, at least one `.idr` file
  under `verification/proofs/`, a generated C header under
  `src/interface/generated/abi/`), or remove them from the File Map and
  add inline caveats that match the ones at the top of the document.
. **Wire a minimal build for Idris2 ABI.** One CI step that runs
  `idris2 --check src/interface/abi/HealthState.idr` is enough to back
  the "formally verified" language in README.adoc. Without it, the
  annotation is aspirational.
. **Wire a minimal `zig build` step.** `src/interface/ffi/build.zig`
  exists; a CI step that runs `zig build` on this path is trivial and
  converts 2 487 LOC of written Zig into built Zig.
. **Fix the TypeScript breach.** Convert `tests/validate.test.ts` to
  either plain JavaScript under Deno or (preferred) ReScript. A test
  harness is still a language-policy subject.
. **Remove or rebadge the "Dogfooded Across The Account" table in
  EXPLAINME** so it cites real integrations (e.g. strike the
  AffineScript↔ASS row unless ASS actually consumes `.as` sources).
. **Rename `.machine_readable/contractiles/lust/` → `intend/`** and
  `Intentfile.a2ml` → `Intendfile.a2ml` per the 2026-04 contractile
  rename. Trivial.
. **Resolve the duplicate at `systems-ecosystem/ambientops/session-sentinel/`**
  — submodule, symlink, or delete. Document in `META.a2ml`.

What blocks promotion to B (after C):

. Zero external targets. For a systemd-tray KDE tool the natural
  diverse-6 are: Fedora KDE, Fedora GNOME (AppIndicator — currently
  NOT supported, per EXPLAINME caveat), openSUSE, Ubuntu, Arch, a
  second author on a distinct machine. The EXPLAINME caveat that
  GNOME/AppIndicator is not supported is itself a diversity blocker:
  five of those six targets would need a second Zig FFI back-end.
. No issues-fed-back pipeline. Even one GitHub issue from a second
  human reporting a tray icon bug on Fedora KDE would seed this.
. A tray tool with `last-run-utc = "never"` has no play-log evidence
  whatsoever; "home-stable" needs at minimum a two-week daemon uptime
  record on the author's own desktop.
